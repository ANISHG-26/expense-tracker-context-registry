# Engineering Standards (POC)

## PR rules
- All code changes require a PR and at least one human review.
- CI checks must pass before merge.
- Link to a ticket or issue in every PR.

## Testing expectations
- Unit tests for logic changes.
- Integration tests for API changes.
- Contract tests for service-to-service boundaries.

## Documentation
- Update API contract docs for any API change.
- Record notable decisions as ADRs.
